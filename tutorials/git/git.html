Step-by-Step Guide to Using Git in the Terminal

Git is a powerful version control tool that helps developers track changes in their codebase and collaborate efficiently. If you are new to Git, this guide will walk you through the basic commands to get started using Git from the terminal.

Step 1: Set Up Your Git Configuration

Before you begin, configure your Git user information. This information will be attached to your commits:

git config --global user.name "Your Name"
git config --global user.email "your_email@example.com"

Step 2: Create a New Repository or Clone an Existing One

You can either initialize a new repository or clone an existing one from a remote server like GitHub.

Create a New Repository:

  1. Navigate to the directory where you want to create your project:
    2. cd path/to/your/project
  2. Initialize a new Git repository:
    3. git init

    This command creates a new .git folder, enabling version control for your project.

Clone an Existing Repository:

If you want to clone an existing repository from a remote server:

git clone <repository-url>

For example:

git clone https://github.com/username/repository.git

Step 3: Add Files to the Repository

You can add new or modified files to the Git staging area.

  1. Create or modify some files in your project.
  2. Use git status to check the status of your files:
    3. git status
  3. Add files to the staging area:
    4. git add <filename>

    Or add all modified files:

    git add .

Step 4: Commit Changes

After staging your changes, commit them to the repository:

git commit -m "Commit message describing changes"

For example:

git commit -m "Added new feature"

Step 5: Connect to a Remote Repository (Optional)

If you haven't already cloned a repository, you can connect your local repository to a remote server (e.g., GitHub).

  1. Add the remote URL:
    2. git remote add origin <repository-url>

    For example:

    git remote add origin https://github.com/username/repository.git
  2. Verify the remote connection:
    3. git remote -v

Step 6: Push Changes to Remote

To send your changes to the remote repository:

git push origin main

If it's your first push, you may need to set the branch:

git push -u origin main

Step 7: Pull Changes from Remote

If others are working on the project, or you are working from different locations, you need to pull the latest changes:

git pull origin main

Step 8: Branching

Branches allow you to work on new features separately from the main codebase.

  1. Create a new branch:
    2. git branch <branch-name>

    For example:

    git branch feature-branch
  2. Switch to the new branch:
    3. git checkout <branch-name>

    Or create and switch simultaneously:

    git checkout -b <branch-name>

Step 9: Merge Branches

When you finish working on a branch, you may want to merge it back into main.

  1. Switch to the main branch:
    2. git checkout main
  2. Merge the changes:
    3. git merge <branch-name>

Step 10: Resolve Merge Conflicts (if any)

Sometimes, merging branches can lead to conflicts. Git will prompt you to resolve these manually.

  1. Open the conflicting files and decide how to merge the changes.
  2. After resolving, add the file:
    3. git add <filename>
  3. Commit the merge:
    4. git commit -m "Resolved merge conflict"

Common Commands Summary

  • View the status of your repository:
    • git status
  • Add all changes to the staging area:
    • git add .
  • Commit staged changes:
    • git commit -m "Your message"
  • View commit history:
    • git log
  • Push changes to the remote repository:
    • git push origin <branch-name>
  • Pull changes from the remote repository:
    • git pull origin <branch-name>

Tips

  • Write meaningful commit messages that describe your changes clearly.
  • Sync frequently using git pull to avoid conflicts.
  • Use branches for developing new features or experiments.
